Every developer is used to test his/her code before deploying it to the production.
But when comes it to test the CI pipelines there is every time the problem how to test it before the deploy to the CI server.
Will be my pipeline working? What does this command and how it affects the others?
Most of the time developers just commit the modification of the pipeline, push it and watch to server how does it working or when it fails.

For the GitHub Actions there is a way how to do this.

## Use Act

To solve this issue you can use [`act`][act] CLI tool to test and develop GitHub Actions locally.
It is based on Docker. You can install it with various ways on different platforms.

Just install it and run `act`. The output will looks like this:

  - Medium size image: ~500MB, includes only necessary tools to bootstrap actions and aims to be compatible with most actions
  - Micro size image: <200MB, contains only NodeJS required to bootstrap actions, doesn't work with all actions

Default image and other options can be changed manually in ~/.actrc (please refer to https://github.com/nektos/act#configuration for additional information about file structure) Micro
